# 💸 Personal Finance Telegram Bot

Многофункциональный Telegram-бот для продвинутого учета личных финансов, написанный на Python (aiogram 3). Бот интегрируется с Google Таблицами, умеет сканировать QR-коды с чеков ФНС, конвертировать валюту и вести учет долгов.

## ✨ Основной функционал

- **Учет расходов и доходов:** Удобное инлайн-меню для категоризации и записи транзакций.
- **Интеграция с Google Sheets:** Все данные надежно хранятся в облаке в виде таблицы.
- **Сканирование чеков (QR):** Автоматическое распознавание QR-кодов с фото, запрос данных через API ФНС и разбивка чека на отдельные позиции товаров.
- **Мультивалютность:** Распознавание символа `$` или `USD` в сообщении и автоматическая конвертация в рубли по актуальному курсу.
- **Трекер долгов:** Учет выданных средств и автоматические напоминания о дедлайнах возврата.
- **Автоматизация подписок:** Планировщик напомнит о дне списания регулярных платежей и предложит внести их в таблицу одним кликом.
- **Аналитика и экспорт:** Генерация круговых диаграмм, расчет кассового остатка, сравнение с прошлым месяцем и выгрузка Excel-отчетов за выбранный в календаре период.
- **Безопасность:** Встроенный Middleware отклоняет любые сообщения от посторонних пользователей (защита по Telegram ID).

## 🛠 Технологии

* **Язык:** Python 3.9+
* **Фреймворк:** `aiogram` 3.x
* **База данных:** Google Sheets API (`gspread`, `oauth2client`)
* **Аналитика:** `pandas`, `matplotlib`
* **Распознавание QR:** `pyzbar`, `Pillow`
* **Автоматизация:** `apscheduler`

## 🚀 Установка и запуск

### 1. Подготовка системы
Для работы библиотеки `pyzbar` (распознавание QR-кодов) требуется системная зависимость `zbar`.
```bash
# Установка системной зависимости (Arch Linux)
sudo pacman -S zbar
```

### 2. Клонирование и зависимости
Клонируйте репозиторий и установите Python-зависимости:
```bash
git clone [https://github.com/yourusername/your-repo-name.git](https://github.com/yourusername/your-repo-name.git)
cd your-repo-name
python -m venv venv
source venv/bin/activate
pip install aiogram gspread oauth2client pandas matplotlib pyzbar Pillow aiohttp apscheduler python-dotenv
```

### 3. Конфигурация
1. Создайте файл `.env` в корне проекта и заполните его по шаблону:
```env
BOT_TOKEN=твой_токен_от_BotFather
SPREADSHEET_ID=айди_гугл_таблицы
USER_ID=твой_телеграм_айди
PROVERKA_CHEKA_TOKEN=токен_от_proverkacheka_com
JSON_KEYFILE=service_account.json
```

2. Поместите файл сервисного аккаунта Google API (`service_account.json`) в корень проекта.
3. Убедитесь, что в вашей Google Таблице есть листы с названиями **Лист 1** (для расходов) и **Доходы**. Сервисному email-адресу из JSON-файла должны быть выданы права "Редактор" на эту таблицу.

### 4. Запуск
```bash
python main.py
```

## 📂 Структура проекта

* `main.py` — Точка входа, запуск бота и планировщика задач.
* `config.py` — Инициализация переменных окружения.
* `handlers.py` — Основная логика обработки сообщений и инлайн-кнопок.
* `gsheets.py` — Класс для взаимодействия с Google Таблицами.
* `keyboards.py` — Генерация Reply и Inline клавиатур, включая календарь.
* `states.py` — Состояния (FSM) для пошагового ввода данных.
* `qr_scanner.py` — Декодирование фото и работа с API чеков.
* `middlewares.py` — Проверка прав доступа пользователя.
* `jobs.py` — Функции для автоматических рассылок (напоминания, долги).
